# SPEC: Obsidian Knowledge Base Integration

**Epic:** `obsidian-integration`
**Created:** 2026-05-26
**Updated:** 2026-05-26
**Status:** Ready for Planning

---

## Goal

Single Obsidian vault inside the repo serves as authoritative project knowledge base.
Every git commit triggers a Claude-powered post-commit hook that writes an AI-generated
summary note to the vault and indexes it into the existing pgvector RAG pipeline — so
the chat app can instantly answer questions about any commit, epic, or architecture
decision without manual uploads.

---

## User Stories / Requirements

### US-01: Commit Auto-Documentation
> As a developer, I want every commit to automatically produce a Markdown note in the
> Obsidian vault with an AI-generated summary, so I never manually write commit docs.

### US-02: Vault RAG Indexing
> As a developer, I want all vault `.md` files indexed into the existing pgvector pipeline
> on app startup, so the chat has full knowledge base context from day one.

### US-03: Chat Answers from Vault
> As a developer, I want to ask the chat questions like "what changed in the auth module
> last week?" and get answers pulled from vault commit notes automatically.

### US-04: Vault Docs Visible in Docs FE
> As a developer, I want to see which vault files are indexed (with their AI-generated
> notes) in the Docs app, so I know the knowledge base is up to date.

## Technical Design

### Vault Structure

```
docs/obsidian-vault/
├── .obsidian/                      # Obsidian app config (git-tracked except workspace state)
├── Index.md                        # Vault root — wikilinks to all sections
├── project/
│   ├── Overview.md                 # What this system is
│   ├── Architecture.md             # Services map: api-gateway, auth-service, ai-service, MFEs
│   ├── Roadmap.md                  # Phase history + current state
│   └── Decisions.md                # ADR-lite: key architecture decisions
├── commits/
│   ├── _MOC.md                     # Auto-updated index of all commit notes (wikilinks)
│   └── YYYY-MM-DD-<hash>.md        # One note per commit (auto-generated by hook)
└── epics/
    └── <epic-name>.md              # Manually maintained epic overview + task status
```

### Data Flow

```
git commit
    │
    ▼
post-commit hook (scripts/obsidian-commit-doc.sh)
    │
    ├─► claude --print <diff>  ──► docs/obsidian-vault/commits/YYYY-MM-DD-<hash>.md
    │
    └─► POST http://localhost:4001/vault/sync { filePath: "docs/obsidian-vault/commits/..." }
                │
                ▼
        VaultSyncService.syncFile(filePath)
                │
                ▼
        DocumentService.uploadDocument(filePath, title, filename)
                │
                ├─► splitIntoChunks()
                ├─► OllamaEmbeddingService.generateEmbedding() per chunk
                └─► INSERT INTO chunks (content, embedding::vector, document_id)
                                │
                                ▼
                        pgvector (768-dim, cosine similarity)
                                │
                                ▼
                    SearchService.similaritySearch(query)
                                │
                                ▼
                        chat conversation context (already wired)
```

### BE: VaultSyncService

**File:** `ai-platform/apps/ai-service/src/vault/vault-sync.service.ts`

```typescript
// On startup: scan docs/obsidian-vault/**/*.md
// For each file: check documents table by filePath
//   - Not found → DocumentService.uploadDocument(filePath, title)
//   - Found but mtime > updatedAt → delete chunks + re-upload
// syncFile(filePath): same logic for single file (called by controller)
```

**Key constraint:** vault docs use `DOCUMENTATION` type (not `GUIDE`) — existing chunking + notes pipeline applies.

**Title derivation:** strip vault prefix + extension → humanized slug.
Example: `docs/obsidian-vault/commits/2026-05-26-abc1234.md` → `"Commit 2026-05-26 abc1234"`

### BE: VaultController

**File:** `ai-platform/apps/ai-service/src/vault/vault.controller.ts`

```
POST /vault/sync    { filePath: string }  → { documentId, chunksCount }
GET  /vault/status                        → { indexed: number, lastSync: ISO8601 }
```

**No auth guard** — internal endpoint, called only from localhost git hook.

### Git Hook Script

**File:** `scripts/obsidian-commit-doc.sh`
**Installed as:** `.git/hooks/post-commit` (symlink)

```bash
#!/usr/bin/env bash
set -euo pipefail

VAULT_DIR="docs/obsidian-vault/commits"
HASH=$(git rev-parse --short HEAD)
DATE=$(date +%Y-%m-%d)
NOTE_FILE="${VAULT_DIR}/${DATE}-${HASH}.md"
SYNC_URL="${VAULT_SYNC_URL:-http://localhost:4001/vault/sync}"

# 1 — Generate note via Claude (fail-safe: skip if claude absent)
if command -v claude &>/dev/null; then
  DIFF=$(git show HEAD --stat --patch | head -300)
  REPO_TAG=""
  echo "$DIFF" | grep -q "ai-platform-fe/" && REPO_TAG="$REPO_TAG fe"
  echo "$DIFF" | grep -q "ai-platform/"    && REPO_TAG="$REPO_TAG be"
  echo "$DIFF" | grep -q "\.planning/"     && REPO_TAG="$REPO_TAG planning"

  claude --print "
You are a technical documentation agent.
Write a concise Markdown note for an Obsidian knowledge base vault.

Output exactly this format (no extra commentary):
---
tags: [commit$(echo $REPO_TAG | sed 's/ /, /g')]
date: ${DATE}
hash: ${HASH}
author: $(git log -1 --pretty=%an HEAD)
---

## Summary
<2-4 sentence plain-language description of what changed and why>

## Changed Files
<bulleted list: filename — one-line description of change>

## Impact
<note if change affects architecture, APIs, DB schema, or agent behaviour — omit section if not applicable>

Git diff:
${DIFF}
" > "${NOTE_FILE}" || true
fi

# 2 — Update _MOC.md
COMMIT_MSG=$(git log -1 --pretty=%s HEAD)
echo "- [[${DATE}-${HASH}]] — ${COMMIT_MSG}" >> "${VAULT_DIR}/_MOC.md"

# 3 — Index new note into RAG pipeline (fail-safe)
if command -v curl &>/dev/null && [ -f "${NOTE_FILE}" ]; then
  curl -sf -X POST "${SYNC_URL}" \
    -H "Content-Type: application/json" \
    -d "{\"filePath\": \"${NOTE_FILE}\"}" \
    > /dev/null || true
fi

exit 0
```

## Out of Scope

| Feature | Reason |
|---------|--------|
| Vector DB migration (pgvector → Qdrant/Weaviate) | pgvector already in prod, no infra change |
| Obsidian Sync / cloud hosting | Local-first; vault is git-tracked |
| Two-way sync (Obsidian edits → re-index) | v1: git commit is the only write trigger |
| Embedding model upgrade (768 → 1536-dim) | Existing Ollama model sufficient for v1 |
| Manual re-index UI button | `GET /vault/status` + startup scan covers it |
| Auth on `/vault/sync` | Internal localhost-only endpoint; git hook has no token |

## Constraints

- `claude --print` must be in PATH (`npm i -g @anthropic-ai/claude-code`)
- Ollama must be running for `VaultSyncService` startup scan to complete
- `VaultSyncService` startup scan is **non-blocking** — ai-service boots even if Ollama is down
- ai-service port `4001` must be reachable from localhost when hook fires
- `DocumentService.uploadDocument` supports `.md` — no changes needed there
- Vault `.md` files are git-tracked; binary attachments forbidden
