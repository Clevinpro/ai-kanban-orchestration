<objective>
Extend kanban-server/index.js with four additions required before any UI code can function:
1. Add X-Accel-Buffering: no to GET /events to fix SSE buffering through the Vite proxy (Pitfall 1 from RESEARCH.md).
2. Add PATCH to the CORS Access-Control-Allow-Methods header (D-06).
3. Add PATCH /tasks/:id/status endpoint that validates taskId format, validates status against VALID_STATUSES, does the regex-replace write (same pattern as the stop endpoint), and returns { ok: true, task } (D-05).
4. Add Express static serving from kanban-server/public/ and a SPA catch-all route after all API routes.

Also update kanban-server/package.json with dev/build/preview scripts that delegate to the client/ subdirectory.

Purpose: Establishes the server-side foundation that the React client (Plan 06-02) and drag-and-drop (Plan 06-03) depend on. Without the PATCH endpoint, drag-and-drop cannot write status back to disk. Without X-Accel-Buffering, SSE events are batched and not streamed live in dev mode.

Output: Modified kanban-server/index.js and kanban-server/package.json.
</objective>
