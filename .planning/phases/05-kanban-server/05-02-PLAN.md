<objective>
Add the stopped-status gate check to execute.md at all four agent invocation points in the pipeline (per D-03). When the kanban server writes status: stopped to a task file, the pipeline must detect it on its next stage-gate read and halt gracefully with a [pipeline] STOPPED receipt — without advancing to the next agent.

Purpose: Without this check, a user who stops a task via the server API would see the pipeline continue running the next agent stage anyway, ignoring the stop signal. This plan delivers the pipeline-halt vertical slice: the moment a stop is written to disk, the pipeline stops at the next gate.
