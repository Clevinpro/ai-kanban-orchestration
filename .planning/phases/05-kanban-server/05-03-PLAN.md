  const fmMatch = content.match(/^---\n([\s\S]*?)\n---/);

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → POST /tasks/:id/stop | Untrusted taskId param crosses here; must be validated before use in file path or shell command |
| server process → sub-repo git | execSync with cwd option; must scope to correct sub-repo dir, never workspace root |
| chokidar → parseTaskFile | File content from disk; malformed YAML must not crash the server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | POST /tasks/:id/stop — taskId path param | mitigate | Validate taskId against /^TASK-\d{3}$/ before constructing file path or branch name; return 400 on mismatch (ASVS V5.1) |
| T-05-06 | Elevation of Privilege | execSync shell injection via taskId | mitigate | Same regex gate as T-05-05; taskId is only 'TASK-' + three digits after validation — no shell metacharacters possible |
| T-05-07 | Tampering | git commit to wrong repo (workspace root) | mitigate | Always pass { cwd: repoDir } to all execSync git calls; repoDir is 'ai-platform' or 'ai-platform-fe' derived from task.repo field |
| T-05-08 | Denial of Service | SSE connection leak on client disconnect | mitigate | req.on('close') registered immediately when client is added; removes res from clients array before next pushEvent call |
| T-05-09 | Information Disclosure | Partial YAML read on rapid file writes | mitigate | chokidar awaitWriteFinish: { stabilityThreshold: 200 } prevents reading mid-write; parseTaskFile returns null on malformed YAML |
| T-05-SC | Tampering | npm install (express, chokidar, js-yaml) | accept | All three packages verified [OK-MANUAL] in RESEARCH.md Package Legitimacy Audit with 100M+ weekly downloads and >10 year histories; no postinstall scripts confirmed |
</threat_model>
