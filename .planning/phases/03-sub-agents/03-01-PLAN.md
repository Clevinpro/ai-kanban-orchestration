<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| system prompt → sub-repo filesystem | Agent instructed to restrict writes to one sub-repo; enforcement is prompt-only (no filesystem sandbox) |
| agent output → orchestrator receipt parser | execute.md greps the final output line for the signal word; malformed output could mislead the orchestrator |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | be-developer system prompt | mitigate | Hard STOP clause (D-02) explicitly names ai-platform-fe/ as forbidden; error receipt [be-developer] ERROR encodes the violation for the orchestrator |
| T-03-01-02 | Tampering | fe-developer system prompt | mitigate | Hard STOP clause names ai-platform/ as forbidden; error receipt [fe-developer] ERROR encodes the violation |
| T-03-01-03 | Spoofing | agent name: field | mitigate | name: must match execute.md subagent_type exactly (hyphens, lowercase); verified by grep "^name: be-developer$" acceptance criterion |
| T-03-01-04 | Elevation of Privilege | tools: array ordering | mitigate | Glob at position 1, Grep at last position — verified by grep -E "^tools: Glob,.+,Grep$" acceptance criterion (bug #60237) |
| T-03-01-05 | Repudiation | receipt protocol | mitigate | Receipt line is the final output — orchestrator can parse it deterministically; agent system prompt instructs exact format |
</threat_model>
