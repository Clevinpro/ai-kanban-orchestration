Create three sub-agent definition files: code-reviewer (read-only denylist agent), qa-be (BE test runner), and qa-fe (FE test runner). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| code-reviewer output → task file (via orchestrator) | code-reviewer cannot write directly; output text is parsed by Phase 4 execute.md for the review block |
| qa agent → task file body | QA agents write directly via Edit; hook validates any status: field in the edit |
| qa agent → sub-repo filesystem | QA agents run Bash in sub-repos; must not run nx outside the sub-repo directory |
| review block Status: field → task-state-guard hook | capital S Status: in body blocks is hook-safe; lowercase status: would trigger validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | code-reviewer disallowedTools | mitigate | disallowedTools: Write, Edit, Bash declaratively blocks all file writes at runtime; verified by grep "^disallowedTools:" and absence of "^tools:" |
| T-03-02-02 | Tampering | QA body annotation — hook evasion | mitigate | System prompt explicitly instructs capitalized Status: (capital S) in ## QA Results block; task-state-guard.js regex is case-sensitive lowercase-only — capital S bypasses hook safely |
| T-03-02-03 | Elevation of Privilege | qa-be running nx outside sub-repo | mitigate | System prompt mandates explicit "cd ai-platform/" before any nx invocation; workspace root has no nx.json — wrong directory fails immediately with "Not an Nx workspace" |
| T-03-02-04 | Elevation of Privilege | qa-fe running nx outside sub-repo | mitigate | Same mitigation as T-03-02-03: "cd ai-platform-fe/" mandated; explicit --base=HEAD~1 --head=HEAD prevents over-wide test scope |
| T-03-02-05 | Spoofing | receipt format mismatch | mitigate | System prompts encode exact receipt strings; execute.md greps for signal word in receipt format [agent-name] SIGNAL |
| T-03-02-06 | Tampering | code-reviewer REVIEW-BLOCK delimiters | mitigate | Delimiters (---REVIEW-BLOCK-START---, ---REVIEW-BLOCK-END---) are unusual enough that task file content is unlikely to contain them naturally; Phase 4 orchestrator parses between these delimiters |
</threat_model>
