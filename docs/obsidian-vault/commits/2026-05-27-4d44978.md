## Summary
Adds `--auto-next` flag to `team-lead:execute` skill so the pipeline automatically chains to the next `readyForDevelop` task in the same epic after completion. The kanban UI gains a persistent auto-run toggle (stored in localStorage) that passes `autoNext: true` to the server on drag-to-inProgress, which appends `--auto-next` to the iTerm invocation. A new `run-task.sh` shell wrapper centralises task launch and reverts status to `readyForDevelop` on unexpected exit.

## Changed Files
- `.claude/commands/team-lead/execute.md` — adds `--auto-next` flag parsing (STEP 1), fixes resume-stage table off-by-one, adds STEP 4 auto-next chain loop
- `kanban-server/client/src/App.jsx` — adds `autoRun` state persisted to localStorage; effect detects newly-done tasks and fires next task PATCH with `autoNext: true`
- `kanban-server/client/src/components/Board.jsx` — accepts `autoRun`/`setAutoRun` props; passes `autoNext` on drag; adds sticky bottom toolbar with toggle and next-task preview
- `kanban-server/index.js` — `PATCH /status` inProgress branch spawns `run-task.sh` via osascript with optional `--auto-next`; deleted-task SSE event now includes `epic` field
- `kanban-server/run-task.sh` — new script: launches `team-lead:execute` with forwarded flags; EXIT trap reverts `inProgress` → `readyForDevelop` on abort

## Impact
**Agent behaviour:** `team-lead:execute` can now chain tasks autonomously without human re-trigger — changes pipeline control flow. **API:** `PATCH /tasks/:epic/:id/status` body accepts new `autoNext` boolean field. **Architecture:** task launch extracted from inline osascript string into `run-task.sh`; all future flag additions go through `FLAGS` forwarding in that script.
